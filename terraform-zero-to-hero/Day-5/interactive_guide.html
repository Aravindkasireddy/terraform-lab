Reference existing infrastructure and manage resource relationships

üìäWhat are Data Sources?

Data sources allow Terraform to fetch information about existing resources that are NOT managed by Terraform. This enables integration with existing infrastructure.

# Query existing VPC
data "aws_vpc" "existing" {
  filter {
    name   = "tag:Name"
    values = ["production-vpc"]
  }
}

# Use the VPC ID in new resources
resource "aws_subnet" "app" {
  vpc_id = data.aws_vpc.existing.id
  cidr_block = "10.0.1.0/24"
}
üí° Data vs Resources

Resources: Create/manage infrastructure ‚Ä¢ Data Sources: Read existing infrastructure

‚ö°Dependencies

Terraform automatically manages resource creation order based on dependencies. You can also explicitly define them when needed.

# Implicit dependency (automatic)
resource "aws_instance" "web" {
  subnet_id = aws_subnet.main.id  # Terraform knows: subnet first
}

# Explicit dependency (when needed)
resource "aws_instance" "app" {
  depends_on = [aws_security_group.allow_ssh]
}
