Manage multiple environments (dev, staging, prod) with a single codebase

üåçWhat are Workspaces?

Terraform workspaces allow you to manage multiple instances of the same infrastructure using a single configuration. Each workspace has its own state file.

# List workspaces
terraform workspace list

# Create new workspace
terraform workspace new dev

# Switch workspace
terraform workspace select prod

# Show current workspace
terraform workspace show

‚öôÔ∏èDynamic Configuration

Use the lookup() function to select different values based on the current workspace.

variable "instance_type" {
  default = {
    "dev"   = "t3.micro"
    "stage" = "t2.medium"
    "prod"  = "t2.xlarge"
  }
}

module "ec2_instance" {
  source        = "./modules/ec2_instance"
  instance_type = lookup(var.instance_type, terraform.workspace, "t2.micro")
}
Workspace Instance Type Use Case
dev t3.micro Development & Testing
stage t2.medium Staging & QA
prod t2.xlarge Production Workloads
